A secure, serverless Digital Certificate Verification Portal built entirely on **Google Apps Script** and **Google Sheets**. This system allows institutions to generate cryptographically secure QR codes for certificates and provides a responsive web portal for instant authentication.

---

## 🎯 Purpose & Audience

**Who is this for?**

This system is built specifically for **college clubs, student organizations, event managers, and workshop coordinators** who need a **free, automated way** to secure and authenticate their awards/Certificates — without paying for expensive enterprise software.
 
> 📌 **Note:** This tool does **not** design the visual certificates. It generates the secure QR codes and the verification web portal to be used **alongside your own certificate designs.

**Why was this created?**

Traditional paper or PDF certificates are easily forged. We created this tool to give students **verifiable proof of their achievements**. When students attach these certificates to their resumes or LinkedIn profiles, **recruiters, HR professionals, and internship coordinators** can simply scan the QR code to instantly authenticate the credential with 100% confidence.
```
Student earns certificate  →  QR code embedded  →  Shared on Resume / LinkedIn
                                                              ↓
                                                    Recruiter scans QR
                                                              ↓
                                                  ✅ Instantly Verified
```
 
> No emails to the organizer. No PDFs to cross-check. No doubt. Just a scan.
***simple and easy***

## ✨ Features

- **Zero-Server Infrastructure** — Runs entirely on Google Workspace (Sheets + Apps Script), no hosting required.
- **Cryptographic Security** — Uses HMAC-SHA256 hashing to generate tamper-proof tokens, preventing URL manipulation.
- **Automated QR Generation** — Automatically generates QR codes directly inside your Google Sheet using the `qrserver` API.
- **Responsive Web Portal** — Clean, mobile-friendly UI to display verified certificate details.
- **Multi-Category Support** — Handles different certificate types (e.g., Participation vs. Organising Committee) across multiple sub-sheets.

## 🚀 How It Works

```
Student Data  →  Token Generation  →  QR Code  →  Certificate
     ↓                                                  ↓
Google Sheet        HMAC-SHA256              Scan QR to Verify
                    Signature                       ↓
                                         Web App validates token
                                                    ↓
                                         ✅ Shows student details
```

1. **Data Entry** — Enter student details into the Google Sheet.
2. **Token Generation** — The script hashes the Certificate ID with a secret key to create a unique signed token.
3. **QR Creation** — A QR code is generated linking to the Web App URL with the secure token embedded.
4. **Verification** — When scanned, the Web App decodes the token, verifies the HMAC signature, and displays the student's certificate data if valid.

## 🗂️ Sheet Structure

The spreadsheet must have two sheets named **`PART`** (Participants) and **`OC`** (Organising Committee), each with the following columns:

| Column | Header | Description |
|--------|--------|-------------|
| A | ID | Unique Certificate ID |
| B | Name | Student Full Name |
| C | PRN | Permanent Registration Number |
| D | Email ID | Student Email Address |
| E | Academic Year | e.g., 2025–2026 |
| F | Event | Event / Workshop Name |
| G | Token | *(Auto-generated by script)* |
| H | QR | *(Auto-generated by script)* |

## 📥 Installation & Setup

Please see the [STARTUP.md](STARTUP.md) file for a complete step-by-step guide on how to deploy this system for your own institution.

### Quick Overview

1. Open your Google Sheet → **Extensions → Apps Script**
2. Paste in `Code.gs` and update the config block at the top
3. Run `generateAllQRs()` to populate tokens and QR codes
4. Deploy as a **Web App** (Execute as: Me, Access: Anyone)
5. Update `scriptUrl` in `Code.gs` with your deployment URL
6. Redeploy with the new version

## ⚠️ Security Warning

- **Never** expose your `SECRET_KEY` in frontend HTML or commit it to a public repository.
- If compromised, bad actors could generate fake valid tokens.
- For production use, store the key in **Script Properties** instead of hardcoding:

```js
const SECRET_KEY = PropertiesService.getScriptProperties().getProperty("SECRET_KEY");
```

Then set it in Apps Script under **Project Settings → Script Properties**.

## 🧪 Local Testing

To test `doGet` from the Apps Script editor without deploying, use the included helper:

```js
function testDoGet() {
  const fakeEvent = {
    parameter: { token: "PASTE_TOKEN_FROM_SHEET_HERE" }
  };
  const result = doGet(fakeEvent);
  Logger.log(result.getContent());
}
```

> ⚠️ Never run `doGet` directly from the editor — it requires a web request context and will throw a `TypeError`.
